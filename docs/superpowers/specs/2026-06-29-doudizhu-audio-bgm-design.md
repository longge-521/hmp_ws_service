# 欢乐斗地主 BGM 音量与曲风优化设计

## 背景

当前前端音频由 `frontend/src/composables/useSoundEngine.ts` 统一管理。大厅和牌局都使用同一套主音量、BGM 音量与音效音量设置：

- 默认主音量为 `0.7`。
- 默认音效音量为 `0.8`。
- 默认 BGM 音量为 `0.35`。

因此大厅 BGM 的实际播放音量约为 `0.35 * 0.7 = 0.245`，明显低于叫地主、出牌等音效的实际音量 `0.8 * 0.7 = 0.56`。这会造成大厅音乐存在感偏弱，而牌局内语音和出牌音效更突出。

同时，现有牌局 BGM 固定为 CDN 上的 `playing.mp3`，没有风格选择和回退策略。若用户不喜欢该曲风，只能关闭音乐，无法在“经典”和“轻快”之间切换。

## 目标

1. 提升外部大厅 BGM 的默认听感，让大厅音乐不再明显小于打牌音效。
2. 为牌局内 BGM 提供更轻快、明亮、循环耐听的默认曲风，风格参考欢乐斗地主常见的活泼棋牌游戏氛围。
3. 保留当前音乐作为经典风格或回退资源，避免新曲源加载失败时出现无音乐。
4. 保持现有开关、主音量、音效音量和 BGM 音量设置语义不变。

## 非目标

- 不直接接入或复制未授权的腾讯欢乐斗地主原始音乐资源。
- 不重做设置弹窗视觉样式。
- 不调整叫地主、出牌、炸弹等音效本身的音量与触发逻辑。
- 不改变后端或 WebSocket 协议。

## 方案

### BGM 配置

在 `useSoundEngine.ts` 中增加集中式 BGM 配置，按场景和风格管理：

- `lobby` 使用大厅曲目，默认场景增益高于牌局。
- `game` 支持 `bright` 和 `classic` 两种风格。
- `bright` 作为牌局默认风格，曲风要求轻快、明亮、循环不刺耳。
- `classic` 保留当前 `playing.mp3`。

建议默认值：

- 大厅场景增益：`1.55`，让默认实际音量接近 `0.38`。
- 牌局场景增益：`1.0`，避免背景音乐压过叫地主和出牌音效。
- BGM 最终音量统一限制在 `0~1`。

最终音量计算：

```text
audio.volume = clamp(masterVolume * bgmVolume * sceneGain, 0, 1)
```

### 音乐风格设置

在声音设置中持久化新增字段：

```ts
bgmStyle: 'bright' | 'classic'
```

兼容旧用户本地配置：读取旧 `hmp_sound_settings` 时，如果没有 `bgmStyle`，默认补为 `bright`。

设置入口可以先保持轻量：音频引擎提供 `setBgmStyle()` 和 `getSettings()` 字段，后续设置面板可以展示“轻快 / 经典”切换。若本次只做引擎能力，也能先默认使用轻快风格。

### 失败回退

`startBgm(name)` 创建 `HTMLAudioElement` 时，为新曲源绑定错误处理：

1. 若当前风格曲源播放失败，且存在回退曲源，则自动切换到经典曲源。
2. 回退仍失败时，只记录警告，不影响游戏流程。
3. 已在播放同一场景和同一风格时，不重复创建音频对象。

### 测试

新增 `useSoundEngine` 单元测试，优先覆盖纯行为：

- 默认大厅 BGM 音量高于默认牌局 BGM 音量。
- 旧设置无 `bgmStyle` 时会补齐默认值 `bright`。
- `setBgmStyle('classic')` 后，牌局 BGM 使用经典曲源。
- 当轻快曲源失败时，牌局 BGM 回退到经典曲源。

为了减少浏览器音频 API 测试复杂度，测试中 mock `Audio` 和 `localStorage`，验证创建的 URL、音量和错误回退行为。

## 验收标准

- 进入大厅后，BGM 默认听感明显比当前版本更清楚，但不盖过按钮音效。
- 进入牌局后，默认 BGM 风格更轻快，叫地主、出牌等关键音效仍然清晰。
- 用户关闭 BGM 后，不会自动播放或回退播放。
- 本地已有声音设置的用户不会因为新增字段丢失原有音量、开关和语音性别设置。
- 前端单元测试通过。
