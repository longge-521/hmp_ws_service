# HMP WS Service 控制台与审计日志优化变更记录 (Update Notes)

为了提供更佳的用户体验、更高的系统性能与防错能力，我们对 HMP WS Service 项目进行了一系列深度的审计日志重构及控制台前端打磨。以下为本次主要更新内容：

---

## 🛡️ 一、 审计日志底层与拦截机制升级

### 1. 物理字段安全扩充 (MySQL)
数据库 `hmp_audit_log` 物理表及 ORM 模型安全扩充了三个新字段：
- `request_params` (text)：记录接口的调用传参，安全规避了 `token` 等敏感属性。
- `execution_time` (float)：接口毫秒级执行耗时。
- `method` (varchar)：请求的 HTTP 方式（例如 `GET` / `POST` / `DELETE` 等）。

### 2. 统一拦截路由设计 (`AuditLogRoute`)
- 移除了原先分散在 Message 路由与 Upload 路由中手写记录日志的旧逻辑。
- 引入了全局、无侵入的 APIRoute 拦截器 `AuditLogRoute`，在请求周期中自动计时、解析 Body/Query/Path 并异步执行入库，保障性能。

---

## ⚡ 二、 Redis 异步滑动防抖去重机制

### 1. 分布式高频防抖（3秒周期）
- 为防止用户在前端连续高频多次点击查询接口导致审计日志泛滥，引入了 Redis 缓存锁定机制。
- 每次查询前将根据操作者、动作名及传参哈希组装唯一指纹 Key，向 Redis 发送 `SET key 1 EX 3 NX` 锁定。
- 重复请求仅返回数据，静默跳过审计日志的重复数据库写入。

### 2. 优雅降级与密码配置
- **自愈式降级**：若本地 Redis 发生宕机或无法连接，系统自动捕获并打印 Warning 警告，直接放行请求并降级记录常规审计，绝不阻断任何主业务流程。
- **端口适配**：检测并修正了 Docker Redis 容器的 `6370` 端口映射配置。

### 3. 滑动过期时间（Sliding Expiration）优化
- 将原有的固定 3 秒过期升级为**滑动窗口过期**：每次命中防抖拦截时，均自动执行 `EXPIRE` 顺延 3 秒。
- **效果**：在用户连续疯狂点击期间，从始至终只会在起始点产生一条审计日志，最大化过滤了高频查询垃圾数据。

---

## 🖥️ 三、 方案 A：全屏沉浸式控制台页面重构

为了提升大屏幕高分辨率下的空间利用率和可读性，对前端页面进行了全平铺改造：
1. **视口 100% 铺满**：
   - 将 body 宽高设为 `100vw` * `100vh`，内边距归零，禁止全局视口溢出滚动。
   - 剥离 `.dashboard` 容器的圆角、外边框与外阴影，将其宽高拉满平铺。
2. **独立高度与局部滚动**：
   - 侧边栏与右侧主操作面板高度设为 `100%`。
   - 右侧主操作面板开启独立 `overflow-y: auto` 溢出滚动，配合精美的半透明美化滚动条，确保页面整体结构固定端正。
3. **组件自适应弹性伸展**：
   - 重构了“WebSocket调试终端” `.terminal` 的高度逻辑为弹性高度 `flex: 1` 且附带 `min-height: 400px`，使其在大屏下自动撑满底部。

---

## ✨ 四、 控制台 UI 与交互精致化打磨

1. **请求参数悬浮气泡预览（Tooltip）**：
   - 对表格行进行了改造，请求参数过长时自动在前端省略号截断。
   - 鼠标指针悬停于参数单元格时，将利用纯 CSS `::after` 伪类自动浮现一个高亮格式化排版（翡翠绿代码色）的多行 JSON 悬浮气泡，方便秒级排查。
2. **表格交互变色与 Hover 呼吸灯**：
   - 为审计日志表格引入了隔行变色（单双行底色交替）。
   - 为 `<tr>` 行增设了平滑的 `background-color` hover 变化，提供舒适的视觉焦点聚拢效果。
3. **Lucide 图标分页按钮**：
   - 将分页“上一页/下一页”纯文字按钮替换为国际化、极简的方向矢量图标。
   - 统一了全局按钮在 `disabled` 状态下的半透明淡暗（opacity `0.35`）和指针禁用属性，增强交互细节。
